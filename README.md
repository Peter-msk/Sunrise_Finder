# Sunrise Finder

> 주소·좌표·지도·공공데이터·현장사진·공간분석을 하나의 프로젝트에서 연결하는 현장조사 및 GIS 업무 지원 도구

## 1. 프로그램 소개

Sunrise Finder는 주소와 좌표를 찾는 단순 검색 프로그램을 넘어, 조사 대상 위치를 수집하고 지도에서 확인하며 이동 경로, 현장사진, 건축물 정보, 공간분석 결과를 하나의 프로젝트로 관리하기 위해 개발되었습니다.

환경영향평가, 현장조사, 시설물 조사, 사업구간 검토, 정온시설 확인, 공공데이터 조회 등에서 반복되는 주소 검색과 지도 확인 업무를 줄이고, 서로 다른 지도·주소·건축·GIS 데이터를 동일한 지점과 공간 기준으로 연결하는 것을 목표로 합니다.

## 2. 개발 목적

1. 주소, 도로명주소, 지번주소와 좌표를 상호 변환하고 정확한 위치를 확인합니다.
2. 여러 조사 지점을 지도에서 관리하고 팀별 이동 경로와 현장조사 정보를 정리합니다.
3. 공공기관의 지도·건축·공간 데이터를 업무용 한글 정보로 제공합니다.
4. SHP, GPKG, DXF, KML 등 사용자 GIS 파일과 온라인 공간정보를 한 지도에서 표시합니다.
5. 사업구간, 버퍼, WFS 형상과 사용자 파일을 이용한 공간분석을 수행합니다.
6. 프로젝트, 최근 작업, API 설정을 업데이트와 분리하여 안전하게 보존합니다.

## 3. 주요 기능

### 주소와 좌표

- 도로명주소와 지번주소 검색
- 주소를 위도·경도 좌표로 변환
- 좌표를 주소로 역변환
- 검색 결과를 지도에서 확인
- 선택한 주소를 조사지점으로 등록
- 여러 주소와 지점의 목록 관리 및 내보내기
- 내부 요청 코드 대신 사용자가 이해할 수 있는 한글 업무정보 표시

### 지도와 이동 경로

- 조사 지점을 지도 마커로 표시
- 지점 선택, 순서 조정, 팀 배정
- 도로 경로와 이동 순서 검토
- 위치별 메모와 현장 정보 관리
- 지도 마커 팝업과 선택 지점 상세 확인
- 다중 지도·경로 공급자를 고려한 구조
- API 키를 지도 JavaScript에 직접 노출하지 않는 로컬 프록시 방식

### 현장사진

- 조사지점과 현장사진 연결
- 사진 미리보기와 지도 마커 연동
- 프로젝트에 등록된 사진만 안전하게 표시
- 원본 파일 경로를 지도 JavaScript에 직접 전달하지 않는 구조
- 최근 프로젝트를 다시 열었을 때 사진과 지점 연결 복원

### 건축HUB 업무 조회

- 선택 주소를 기준으로 건축 관련 공식 자료 조회
- 기본개요부터 지역지구구역까지 10종 업무정보 지원
- 내부 코드와 요청 파라미터를 숨기고 한글 필드명 중심으로 표시
- 다건 결과의 화면 확인과 CSV 내보내기
- 직접 조회 결과를 자동으로 지점에 추가하지 않고 사용자 선택 시에만 등록

### VWorld 지도 레이어

- VWorld WMS/WFS API 2.0 카탈로그 169종 관리
- 25개 분류별 레이어 검색과 선택
- WMS 표시용 레이어와 WFS 분석용 형상 구분
- 선택 레이어를 목록 상단으로 이동하여 빠르게 관리
- 복수 레이어와 공식 스타일 정보 유지
- 기본지도, 주제도, 분석 결과를 하나의 지도 좌표 프레임에 표시

### 사용자 GIS 파일

- SHP
- GeoPackage(GPKG)
- DXF
- KML
- GeoJSON 등 확장 가능한 벡터 데이터

파일의 좌표계를 확인하여 지도 표시용 WGS84로 변환하고, 실제 거리·면적·버퍼 분석은 국내 업무 좌표계인 EPSG:5179를 중심으로 수행하는 방향을 따릅니다.

### 공간분석

- 사업구간 또는 사용자 선택 범위 등록
- 내부·외부·내외부 분석범위 설정
- 거리 버퍼 생성
- WFS 및 사용자 GIS 레이어와의 교차·포함 관계 분석
- 분석 결과 형상 클리핑
- 사업구간, 버퍼, 분석 결과 레이어의 표시·숨김·제거
- 결과를 프로젝트에 저장하고 재열기
- 표시용 지도와 분석용 좌표계를 분리하여 정확도와 시각화를 함께 유지

### 프로젝트 관리

- 조사 지점, 주소, 경로, 사진, 공간분석 설정과 결과 저장
- 최근 프로젝트 목록
- 프로젝트 다시 열기와 상태 복원
- 사용자 설정과 API 키 보존
- CSV 등 업무용 결과 내보내기
- 업데이트 시 프로젝트와 사용자 데이터를 덮어쓰지 않는 구조

## 4. 기본 업무 흐름

```text
프로젝트 생성
→ 주소·좌표 또는 GIS 파일 불러오기
→ 조사 지점 확인·편집
→ 지도와 경로 검토
→ 건축HUB·VWorld 등 공공데이터 조회
→ 사업구간·버퍼·레이어 공간분석
→ 현장사진과 메모 연결
→ 프로젝트 저장 및 결과 내보내기
```

## 5. 데이터와 보안 원칙

- 실제 API 키를 소스, 배포 ZIP, 로그에 포함하지 않습니다.
- API 키 원문을 JavaScript와 사용자용 로그에 노출하지 않습니다.
- 로그인, 인증, 캡차, 민원 발급 버튼 자동화 기능은 포함하지 않습니다.
- 사용자 승인 없는 다운로드, 외부 전송, 관리자 권한 상승을 수행하지 않습니다.
- 온라인 지도와 API가 필요한 기능 외에는 프로젝트 데이터를 로컬에서 관리합니다.
- 직접 조회한 주소와 건축정보는 사용자가 승인한 경우에만 조사지점으로 등록합니다.

## 6. 개발 방향

Sunrise Finder는 개별 검색 기능의 집합이 아니라, 현장조사와 공간검토의 통합 프로젝트 플랫폼으로 발전하는 것을 목표로 합니다.

- VWorld, 건축HUB 및 다른 공공기관 WMS·WFS·REST·GeoJSON 공급자 확장
- 모든 온라인·오프라인 공간자료를 동일한 렌더링 계약으로 표시
- 주소·경로·사진·건축정보·공간분석 결과의 통합 지점 모델 강화
- 대형 SHP·GPKG·DXF 파일의 로딩과 렌더링 성능 개선
- 지도 캐시와 로컬 프록시의 안정성 및 종료 정리 강화
- 좌표계 자동 판정과 잘못된 CRS 입력에 대한 사용자 안내 개선
- 버퍼, 클리핑, 포함관계, 거리·면적 분석 기능 확장
- 환경영향평가와 현장조사 업무에 맞는 결과표·보고서·레이어 내보내기 강화
- 모바일 현장 앱과 데스크톱 프로젝트 간 데이터 호환 구조 발전
- API 공급자 장애 시 원인과 대체 작업을 명확히 안내하는 진단 체계 강화
- onedir 패키지 전체를 안전하게 교체하면서 API 키와 프로젝트를 보존하는 업데이트 체계 유지

## 7. 지원 환경과 배포

- 대상 운영체제: Windows 10/11 64비트
- 기본 배포 형태: PyInstaller onedir 기반 실행형 ZIP
- 현재 공개 README 작성 기준: Sunrise Finder v2.0.0
- 온라인 지도와 공공데이터 기능은 사용자가 발급받은 각 서비스 API 키와 서비스 제공 상태에 따라 달라질 수 있습니다.

## 프로젝트 공통 원칙

Sunrise Group의 프로그램은 단순한 일회성 스크립트가 아니라 실제 업무에서 반복 사용할 수 있는 Windows 데스크톱 도구를 목표로 합니다.

- 원본 파일과 사용자 데이터를 우선적으로 보호합니다.
- 삭제, 다운로드, 업데이트, 외부 프로그램 실행 등 중요한 동작은 사용자가 인지하고 승인한 경우에만 수행합니다.
- 사용자 문서, 이미지, 프로젝트 데이터와 API 키를 불필요하게 외부로 전송하지 않습니다.
- 장시간 작업은 진행 상태와 현재 처리 단계를 표시하고, 오류 발생 시 사용자용 안내와 진단 로그를 분리합니다.
- 공식 배포본은 제품 정보, SHA-256 무결성 확인, 코드 서명, AhnLab/V3 사전점검 등 Sunrise Group의 배포 기준을 따릅니다.
- 기능 추가보다 기존 기능의 재현성, 복구 가능성, 대용량 처리 안정성, 유지보수성을 우선합니다.
